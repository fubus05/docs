MyAffiliates uses OAuth2 client credentials for authentication and returns affiliate stats as a CSV file. GorAff fetches the CSV over HTTP after obtaining an OAuth2 bearer token.

## Metrics imported

| GorAff field | MyAffiliates CSV column |
|-----------------|-------------------|
| Clicks | `Clicks` |
| Leads | `Signups` or `NRC` |
| Conversions | `FTD Count` or `NDC` |
| Revenue | `Income` |
| rawData.impressions | `Impressions` |
| rawData.qualifiedPlayers | `Qualified Players` |
| rawData.deposits | `Total Deposits` or `Deposits` |
| rawData.ftdAmount | `FTD Amount` |
| rawData.totalBets | `Total Bets` |
| rawData.totalWinnings | `Total Winnings` |

GorAff handles both column naming conventions (`Signups`/`NRC`, `FTD Count`/`NDC`, etc.) across different panel versions.

## Get your credentials

You need to create an OAuth2 client in your MyAffiliates panel to get a **Client ID** and **Client Secret**.

<AccordionGroup>
  <Accordion title="Step 1 — Open Account settings">
    Log in to your MyAffiliates panel and click **Account** in the top navigation bar.

    ![Account menu in the top navigation bar](/images/myaffiliates/01-account-menu.png)
  </Accordion>

  <Accordion title="Step 2 — Go to Authorisation">
    In the left sidebar under **Account**, click **Authorisation**.

    ![Authorisation link in the left sidebar](/images/myaffiliates/02-authorisation-menu.png)
  </Accordion>

  <Accordion title="Step 3 — Create a new OAuth2 client">
    Scroll to the **OAuth2 Clients** section. Enter a name for your client (e.g. `GorAff`) in the **Client name** field, then click **Create new client**.

    ![Create new OAuth2 client form](/images/myaffiliates/03-create-oauth-client.png)
  </Accordion>

  <Accordion title="Step 4 — Client created successfully">
    You will see a success message: **"Your client has been created"**. The page will redirect automatically.

    ![Success message after client creation](/images/myaffiliates/04-client-created.png)
  </Accordion>

  <Accordion title="Step 5 — Copy Client identifier and Client secret">
    After the redirect, you are back on the Authorisation page. The new client is shown with its **Client identifier** and **Client secret**.

    ![Client identifier and Client secret highlighted](/images/myaffiliates/05-copy-credentials.png)

    <Warning>
      Copy the **Client secret** immediately — it will not be shown again after you leave this page.
    </Warning>

    Use these values as `apiUsername` (Client identifier) and `apiPassword` (Client secret) when configuring the partner in GorAff.
  </Accordion>
</AccordionGroup>

## Enable OAuth download for your report

Before connecting, make sure the CSV report is allowed to be downloaded via OAuth:

1. In the panel, go to **Account → Authorisation**
2. Find the report you want to sync
3. Under **Download options**, enable OAuth-based download

If this step is skipped, GorAff will receive an HTML login page instead of a CSV and the sync will fail with: *"Returned an HTML page instead of CSV"*.

## Configure the partner

```json
{
  "name": "MyAffiliates",
  "slug": "myaffiliates",
  "apiUrl": "https://panel.example.com/api/report.csv",
  "authMethod": "OAUTH2_CLIENT_CREDENTIALS",
  "apiUsername": "your-client-identifier",
  "apiPassword": "your-client-secret",
  "apiConfig": {
    "dateFromField": "start_date",
    "dateToField": "end_date",
    "responseType": "throne_csv"
  }
}
```

<Note>
Replace `apiUrl`, `apiUsername`, and `apiPassword` with the values from your panel. The `oauthTokenUrl` defaults to `{origin}/oauth/access_token` — add it to `apiConfig` if your panel uses a different token URL.
</Note>

## How campaigns are created

Each unique `Channel` value in the CSV becomes a campaign. Rows are grouped by `Channel + Date` — multiple pay-period blocks in the same CSV (re-printed headers for each period) are deduplicated automatically.

## Alternative: session cookie

If your panel version doesn't support OAuth download, you can authenticate with a browser session cookie instead:

```json
{
  "authMethod": "CUSTOM_HEADERS",
  "apiConfig": {
    "customHeaders": {
      "Cookie": "your-session-cookie-here"
    },
    "dateFromField": "start_date",
    "dateToField": "end_date",
    "responseType": "throne_csv"
  }
}
```

<Warning>
Session cookies expire. You'll need to update the `customHeaders` cookie value manually whenever it expires. For stable integrations, use OAuth2.
</Warning>

## Troubleshooting

**HTML page returned instead of CSV** — OAuth download is not enabled in the panel. Follow the steps in [Enable OAuth download](#enable-oauth-download-for-your-report).

**`0` recordsProcessed** — the date range returned no rows with a valid `Channel` and `Date`. Verify that the report URL generates data for the requested period.

**Token endpoint error** — add `oauthTokenUrl` to `apiConfig` if the default (`{origin}/oauth/access_token`) doesn't match your panel's endpoint.
