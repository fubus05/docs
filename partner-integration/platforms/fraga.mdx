Fraga Partners uses a REST API that returns JSON with a `data` array. Each row is keyed by `serial_id` (tracker/campaign) and `transaction_date`.

## Metrics imported

| GorAff field | Fraga Partners field |
|-----------------|-------------|
| Leads | `registration` |
| Conversions | `ftd_count` |
| Revenue | `profit` |
| rawData.cpa\_count | `cpa_count` |
| rawData.cpa\_profit | `cpa_profit` |
| rawData.revenue\_share\_profit | `revenue_share_profit` |
| rawData.deposits | `deposits` |
| rawData.ngr | `ngr` |
| rawData.ggr | `ggr` |
| rawData.turnover | `turnover` |

## Get your API link

Fraga Partners uses a pre-authenticated URL as the integration endpoint â€” the token is embedded in the link itself. No separate API key is needed.

<AccordionGroup>
  <Accordion title="How to get your Statistics Retrieval API link">
    1. Log in to your [Fraga Partners](https://fragapartners.com) affiliate panel
    2. In the left sidebar, click **Tracking links**
    3. Go to **Statistics retrieval API**
    4. The full API link will be displayed in the **Statistics retrieval link** field
    5. Click **COPY LINK** to copy it to your clipboard

    ![Statistics Retrieval API page in Fraga Partners panel](/images/fraga/statistics-retrieval-api.png)

    The copied link will look like this:
    ```
    https://api.fragapartners.com/data/affiliate_statistics/reports?token=YOUR_TOKEN&start=DATE&end=DATE&per_page=10&page=1
    ```

    <Note>
      Use only the base part of the URL (up to and including `token=YOUR_TOKEN`). GorAff appends `start`, `end`, `per_page`, and `page` automatically during each sync.
    </Note>
  </Accordion>
</AccordionGroup>

## Configure the partner

```json
{
  "name": "Fraga Partners",
  "slug": "fraga",
  "apiUrl": "https://api.fragapartners.com/data/affiliate_statistics/reports?token=YOUR_TOKEN",
  "authMethod": "NO_AUTH",
  "apiConfig": {
    "dateFromField": "start",
    "dateToField": "end",
    "responseType": "fraga_stats"
  }
}
```

<Note>
  Replace `YOUR_TOKEN` with the token value from the Statistics Retrieval API link you copied in the step above.
</Note>

## How campaigns are created

GorAff uses `serial_id` as the `externalId` for each campaign. The campaign name is taken from `tracker_name`. If multiple rows share the same `serial_id` and `transaction_date`, their metrics are summed before writing.

## Test the connection

After saving the partner, click **Sync now**. If the sync log shows `SUCCESS` and `recordsProcessed > 0`, the integration is working.

If you see `0 recordsProcessed` but the sync succeeded, the API returned data but no rows had a valid `serial_id` and `transaction_date`. Check that the date range includes days with activity.
