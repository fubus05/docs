Affilka returns stats as JSON. Each row can be a flat object or an array of `{ name, value, type }` cells — GorAff handles both formats. Rows are grouped by `campaign_id + date`.

## Metrics imported

| GorAff field | Affilka field |
|-----------------|---------------|
| Clicks | `visits_count` |
| Leads | `registrations_count` |
| Conversions | `first_deposits_count` |
| Revenue | `partner_income` or `clean_net_revenue` |
| rawData.deposits | `deposits_sum` |
| rawData.ngr | `ngr` |

## Get your credentials

Your API key in Affilka is called **Statistic token** and is found in your account info.

<AccordionGroup>
  <Accordion title="Step 1 — Open My account → Account info">
    In the left sidebar, click **My account** to expand it, then click **Account info**.

    ![My account menu expanded in the sidebar](/images/affilka/01-my-account-menu.png)
  </Accordion>

  <Accordion title="Step 2 — Copy your Statistic token">
    On the **Account info** page, find the **Statistic token** field in the **General information** section. Copy the value — this is your API key for GorAff.

    ![Statistic token field in Account info](/images/affilka/02-statistic-token.png)

    <Note>
      If the token field is empty, click **Expire** next to it to generate a new token.
    </Note>
  </Accordion>
</AccordionGroup>

## Configure the partner

```json
{
  "name": "Affilka",
  "slug": "affilka",
  "apiUrl": "https://api.affilka.com/v1/stats",
  "authMethod": "API_KEY",
  "apiKey": "your-affilka-api-key",
  "apiConfig": {
    "dateFromField": "date_from",
    "dateToField": "date_to",
    "responseType": "affilka"
  }
}
```

<Note>
If Affilka's API uses a different parameter name for dates or requires the key in a query string instead of a header, refer to the [apiConfig reference](/partner-integration/api-config) for the relevant options (`dateFromField`, `dateToField`, `tokenField`).
</Note>

## Response shape

GorAff accepts two response shapes from Affilka:

**Shape 1:**
```json
{ "rows": { "data": [ ... ] } }
```

**Shape 2:**
```json
{ "data": [ ... ] }
```

Within each row, the data can be a flat object:
```json
{ "campaign_id": "42", "date": "2026-06-01", "registrations_count": 5 }
```

Or a cell array:
```json
[
  { "name": "campaign_id", "value": "42", "type": "string" },
  { "name": "date", "value": "2026-06-01", "type": "date" }
]
```

Both are normalised into the same internal representation.

## How campaigns are created

Each unique `campaign_id` (or `campaign`) value becomes a campaign. The campaign name defaults to the `campaign_id` — Affilka doesn't always return a human-readable name.

## Test the connection

After saving, trigger a manual sync. Check the sync log for `SUCCESS` and `recordsProcessed > 0`. If you see `0` processed rows, confirm the date range overlaps with active campaign activity.
